1.     Reports

There are a total of 15 reports divided into three main categories:

·         System Reports

·         Input Reports

·         Output Reports

Both the Administrator and the data entry staff have access to reports. To access the reports, mouse-over the Report menu. Choose from the menu and click on the reports needed. Depending on which report is selected, more menus and report options may be shown. Simply select and click on what is required.

All reports are in Microsoft® Excel format. This gives users of the reports the flexibility to process and analyse the data further. It also allows attachments of the reports to any form of electronic mail system facilitating ease of distribution. The reports can also be inserted directly into Microsoft® PowerPoint for presentation.

With the Microsoft® Excel format, the default graph formats can be changed to any other graph types by using the graph wizard option.

Note: Before printing reports, it is recommended that the Administrator previews and touches up the look and feel of the reports in case of pagination problems.

2.     System Reports

System Reports are essentially libraries that are embedded within the software. There is 1 report available in this category.

2.1. List of Competencies

To view the list of competencies, go to Report à System Reports à List of Competencies.

There are a total of three options to choose from. They are as follows:

v     List of Competency

This lists the entire library of competencies available in the system.

v     List of Competency with Definitions

This report is similar to the previous one with the addition of definition for each competency.

v     List of Competency with Definitions and Key Behaviours

This report is similar to the previous one with the addition of key behaviours for each competency.

Click on the Preview button, and the system will generate a Microsoft® Excel file to be downloaded.

3.     Input Reports

Input reports capture all the data that are fed into the system. These reports are useful for further analysis and also act as an audit trail. It also includes analysis results that are obtained from the input. There are a total of 7 reports available in this category.

3.1. Competency Listing for Specific Survey

To view the list of competencies by survey, go to Report à Input Reports à Competency by Survey.

There are a total of three options to choose from. They are as follows:

v     Competency Listing for Specific Survey

This lists all the competencies selected for a chosen survey.

v     Competency Listing for Specific Survey with Definition

This is similar to the above with the addition of definition for each Competency.

v     Competency Listing for Specific Survey with Definitions and Key Behaviours

This is similar to the above with the addition of Key Behaviours for each Competency.

Click on the Preview button, and the system will generate a Microsoft® Excel file to be downloaded.

4.     Output Reports

Output Reports are the most important deliverables the Administrator can obtain from the application. There are a total of 3 reports from this category.

4.1.    Questionnaire Report

To view questionnaire report, go to Report à Output Reports à Questionnaire.

Click on the Preview button, and the system will generate a Microsoft® Excel file to be downloaded.

Figure 9. Questionnaire Report

4.2.    Individual Report

To view individual report, go to Report à Output Reports à Individual Report

This is a collated report intended for the target. It acts as a feedback tool and contains the following section:

·         Executive Summary Report – Summary of rating given to target versus required level presented graphically and through a table highlighting strength and developmental gaps

·         Normative Report – Provide a relative comparison between target and the rest of the target group

·         Individual Profile Report – Rating details for all the competencies assessed

Figure 10. Individual Report

Check on the Simplified Report to generate a report without the Competency and Key Behaviour charts in order to cut down generation process time.

4.3.    Group Report

To view group report, go to Report à Output Reports à Group Report

This report is a collation of all the results of the targets from a survey. The report is intended for senior management, offering a holistic and bird-eye view of the group. The report contains:

·         Executive Summary – Summarising the Targeted Group’s profile based on all the rating input against the required profile. This is presented graphically and also through a table indicating strengths and development gaps

·         Group Competency Profile Report – Rating details for all the competencies assessed

·         Group Competency Distribution Report – Provides information on the distribution of the scores for each competency assessed for the targeted group. This indicates the performance bell curve for the group for each of the competency assessed.

·         Group Ranking Report – Indicates the ranked order of the targeted group based on how they performed in the assessment. Those that has the most and biggest strengths would ranked well in the survey

Figure 11. Group Report

Check Simplified Report if the user would like to generate a report with fewer charts in order to cut down generation process time.

4.4.    Development Guide

To view development guide, go to Report à Output Reports à Development Guide

Based on the gap analysis, all competencies that fall under the category of development area would then have a list of development resources being generated in the guide where the target could use to help them in the post assessment follow up.
